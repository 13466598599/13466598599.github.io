Git分支管理

2019-05-30 评论

为啥要使用Git分支呢?

  • 一般情况,master主分支不做开发,master主分支只做生成环境.
  • 开发一般在子分支, 因为我们在开发过程中,代码避免出现Bug或修改部分代码, 这些问题是在子分支上解决, 解决Bug后,没有任何问题,合并到master主分支上

分支命令

查看所有的分支: git branch -r

.* 是当前所在的分支

查看当前所在的分支: git branch

切换分支: git checkout ‘分支名’

查看提交历史: git log

回退到指定版本: git reset -hard 日志id

查看当前分支下的状态: git status

合并分支

  • 首先进入到master分支下 (gie checkout master)

  • 合并: git merge 分支名

  • 推送: git push

  • 素质三连

  • git merge特点:
    • 只处理一次冲突
    • 只会记录合并的历史记录, 合并的commit 按照提交时间排列展示
